# macOS 초기 개발환경 세팅

> 작성일: 2026-04-25  
> 환경: MacBook (Apple Silicon)  
> 배경: 맥북 초기화 후 풀스택 개발용 세팅 첫 진행

## TL;DR

- Claude Code CLI는 **네이티브 설치**(`install.sh`) 권장 — npm 방식은 deprecated
- Homebrew 설치 직후 **PATH 추가 필수** (Apple Silicon)
- Windows와 명령어 체계 다름 (`pbcopy`, `brew` 등)
- 설치 직후엔 **새 터미널 창**을 열어야 PATH 반영됨

---

## 1. 시작 환경

맥북 초기화 직후 다음만 설치된 상태였다.

- VS Code
- Anaconda
- Claude Desktop

여기에 풀스택 개발에 필요한 환경을 추가로 세팅했다.

Windows에서 풀스택 환경을 세팅해본 경험은 있지만 macOS는 처음이라, 명령어 체계와 경로 구조가 달라 새로 정리할 필요가 있었다.

## 2. Claude Code CLI 설치

### 2.1 설치 방법 선택

2026년 기준 세 가지 방법이 있다.

| 방법 | 명령어 | 비고 |
|------|--------|------|
| **네이티브** | `curl -fsSL https://claude.ai/install.sh \| bash` | 공식 권장, 자동 업데이트 |
| Homebrew | `brew install --cask claude-code` | 수동 업데이트 필요 |
| npm | `npm install -g @anthropic-ai/claude-code` | **Deprecated** |

Windows에서 npm 버전과 네이티브 버전이 충돌해 PATH 문제로 고생한 경험이 있어, 이번엔 네이티브 설치로 진행했다.

### 2.2 설치 명령어

\`\`\`bash
curl -fsSL https://claude.ai/install.sh | bash
\`\`\`

### 2.3 주의사항

- 설치 후 **반드시 새 터미널 창을 열어야** `claude` 명령이 인식됨 (PATH 반영)
- `sudo` 사용 금지 — 권한이 꼬여 오히려 문제 발생
- 무료 Claude.ai 플랜은 Claude Code 미지원, 유료(Pro 이상) 필요

### 2.4 설치 확인

\`\`\`bash
claude --version
\`\`\`

## 3. Homebrew 설치

Homebrew는 macOS의 패키지 매니저로, Windows의 `winget`이나 `choco`에 해당한다. 앞으로 설치할 도구들을 한 곳에서 관리할 수 있어 처음에 설치해두기로 했다.

### 3.1 설치 명령어

\`\`\`bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
\`\`\`

설치 중 비밀번호를 묻는데, 입력 시 화면에 표시되지 않는 게 정상이다 (Windows와 다른 점).

### 3.2 ⚠️ 겪은 에러: brew: command not found

설치가 끝난 직후 `brew --version`을 쳤더니 명령어를 찾을 수 없다는 에러가 발생했다.

\`\`\`bash
brew --version
# zsh: command not found: brew
\`\`\`

설치 자체는 성공했는데 zsh가 brew 실행 파일 위치를 모르는 상태였다. Apple Silicon 맥에서 Homebrew는 `/opt/homebrew/bin/`에 설치되는데, 이 경로가 PATH에 자동 추가되지 않은 게 원인이었다.

### 3.3 해결: PATH 추가

\`\`\`bash
echo >> ~/.zprofile
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
\`\`\`

설치 직후 안내 메시지에 `Next steps`로 이 명령어가 포함되어 있는데, 처음엔 무심코 넘기고 다른 채팅을 통해 다시 확인 후 적용했다.

### 3.4 확인

\`\`\`bash
brew --version
\`\`\`

## 4. Windows vs macOS 차이 정리

세팅하면서 가장 많이 헷갈렸던 부분들이다.

| 항목 | Windows | macOS |
|------|---------|-------|
| 패키지 매니저 | winget, choco | brew |
| 클립보드 복사 | `clip` | `pbcopy` |
| 기본 shell | PowerShell, CMD | zsh |
| 경로 구분자 | `\` | `/` |
| 홈 디렉토리 표기 | `C:\Users\이름` | `~` 또는 `/Users/이름` |
| 환경변수 설정 | 시스템 속성 GUI | `~/.zprofile`, `~/.zshrc` |
| Claude Code 설치 | `irm` 스크립트 | `curl` 스크립트 |
| 줄바꿈 문자 | CRLF (`\r\n`) | LF (`\n`) |
| 파일시스템 대소문자 | 구분 X (NTFS) | 구분 X (APFS) |

특히 **클립보드 복사**가 자주 헷갈렸다. SSH 공개키를 복사할 때 Windows 습관으로 `clip`을 쳤다가 안 되는 걸 발견하고 `pbcopy`로 수정했다.

## 5. 회고

### 잘 된 것

- Windows에서 npm/native 설치 충돌을 겪은 경험을 살려 처음부터 네이티브 설치 선택
- Homebrew를 초기에 설치해 이후 도구 관리 일원화 기반 마련

### 막혔던 것

- 비밀번호 입력 시 화면에 안 보이는 게 처음엔 불안 — Windows와 다른 점 적응 필요
- Homebrew 설치 후 `Next steps` 안내를 무심코 넘겨서 PATH 설정 누락 → `command not found` 에러
